Simple Ajax
Promises-based, with streaming, piping, and automatic body-parsing.

GET('https://myhost.com', { query: 'params' })
  .Accept('json')
  .then(function(res) {
  	res.status; // => 200
  	res.reason; // => 'OK'
    res.ContentType; // => 'application/json'
    res.body; // => { hello: 'world' }
  });

Virtual servers on the page
An emulation of HTTP lets you answer requests from client-side javascript.

local.at('#hi', function(req, res) {
  res.status(200, 'OK');
  res.ContentType('html');
  res.Link({ href: '#hi', rel: 'self service', title: 'Hello Service' });
  res.end('<h1>Hello, '+req.params.who+'</h1>');
});
GET('#hi', { who: 'World' }).then(...);
  • The hash-path #hi is like a traditional single-page-app URL, but it handles Ajax requests instead of just page-navigations.

Virtual servers in Web workers
Workers run in parallel threads with sandboxed VMs, allowing you to load programs from other domains.

local.spawnWorker('/my-worker.js');
GET('/my-worker.js', { who: 'Worker' }).then(function(res) {
  	res.status; // => 200
  	res.reason; // => 'OK'
    res.ContentType; // => 'text/html'
    res.body; // => <h1>Hello, Worker</ht>
});
